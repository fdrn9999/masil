## 1. 한 줄 컨셉

> 오픈채팅으로 가볍게 시작한 썸들이, 결국 "진심이란 무엇인가"를 묻게 되는 홍대 청춘 연애극.

플레이어는 20대 중반 사회초년생이 되어, 오픈챗에서 여러 여성과 썸·연애를 시작한다.
가볍게 어장처럼 굴릴 수도, 한 사람에게 진심을 줄 수도 있다.
그 선택이 **진심 게이지**로 쌓이고, 엔딩을 가른다.

---

## 2. 핵심 차별점 (이 게임만의 정체성)

1. **카톡/오픈챗 메시지 UI** — 대화의 상당 부분이 채팅창 형태. 프사·말풍선 색·말투로 인물 구분.
2. **호감도 + 진심 게이지 이중 수치** — 호감도만 높고 진심이 낮으면 "어장 엔딩". 주제를 시스템으로 표현.
3. **조력자 상담 시스템** — 매 챕터 동생(도윤)에게 카톡 상담 → 힌트 또는 정곡을 찌르는 말. 챕터당 1회 제한으로 전략성.

## 3. 등장인물

### 주인공 (플레이어)
- 20대 중반, 잠실 근처 자취. 사회초년생(또는 복학생). 이름 입력 가능.
- 연애에 서툴고 진심 주는 걸 두려워하는 성격 → 게임 내내 변화하는 축.

### 도윤 (조력자 / 페이몬 포지션) — 21세
- 오픈챗 게임방에서 만남. 처음엔 보이스룸으로 게임만 같이 하던 사이.
- 현실에서 처음 만난 날, 술에 취해 세상과 과거에 분노를 쏟아내며 무너진 도윤을 주인공이 제압하고 집까지 데려다줌. 그 밤 이후로 주인공의 **절대적 지지자이자 조력자**가 됨.
- 성격: 능청스럽고 입은 가볍지만 의리 깊음. 주인공의 회피·어장 습관을 유일하게 직설적으로 찌르는 양심.
- 역할: 힌트 제공, 메시지 해석, 작전 조언 + 후반 감정선의 거울.
- 후반 떡밥: 도윤의 분노("과거", "그 사람")가 어느 히로인 루트와 연결되도록 복선.

### 히로인 A — 서아 (24세) · "빠르게 불타는 상대"
- 첫 만남: 신촌권 대학 연합 오픈챗(같은 학교 또는 근처 대학생). 캠퍼스·학식·과 모임 분위기에서 빠르게 가까워짐. 만난 지 며칠 만에 직진.
- 매력: 화끈하고 솔직, 텐션 높음. 쉬워 보이지만 진심 게이지를 안 쌓으면 가장 빨리 식음.
- 주제: "빠른 연애의 유통기한".

### 히로인 B — 지우 (23세) · "천천히 진짜가 되는 상대" (정통 루트)
- 첫 만남: 신촌 보드게임카페 오픈챗. 처음엔 그냥 게임 친구.
- 매력: 잔잔하고 다정, 느리지만 깊음. 보상이 가장 큰 메인 루트.
- 주제: "친구에서 연인으로 넘어가는 용기".

### 히로인 C — 민결 (25세) · "싸우다 친해진 반전 상대"
- 첫 만남: 오픈챗에서 말다툼으로 시작. 후반부 합류.
- 매력: 날카롭고 거리감 있지만 속이 깊음. 게임의 주제가 터지는 반전 루트.
- 도윤의 과거와 얽히는 인물로 설계 가능 (후반 한 방).

> 인물 구분 장치: 각자 **첫 등장 맥락을 다르게** (서아=대학 연합 오픈챗/캠퍼스, 지우=보드게임카페/게임, 민결=오픈챗 말다툼). 입구가 다르면 플레이어 머릿속에서 자동 분류됨. 카톡 UI의 프사·말풍선 색·말투도 확실히 구분.

## 4. 맵 = 서울 지하철 2호선 (순환선)

맵 이동의 큰 틀을 **2호선 순환선**으로 잡는다. 플레이어는 노선도 화면에서 역을 골라 이동하고,
각 역은 고유한 분위기를 가진 **에피소드 무대(허브)**가 된다.

```
        ─ 홍대입구 ─ 합정 ─ 당산 ─ 문래 ─ 신도림 ─
       /                                          \
   신촌                                            대림
     │            ◀  2호선 순환선  ▶                 │
   이대                                            구로디지털
     │                                              │
   시청 ─ 을지로 ─ 성수 ─ 건대입구 ─ 잠실 ─ 강남 ─ 서초 ─
```

### 역별 무대 설정

| 역 | 분위기 | 주 용도 |
|---|---|---|
| **잠실** | 롯데타워·석촌호수·자취방 | **주인공 거점**, 일상·오픈챗 베이스, 큰 이벤트 데이트 |
| **신촌·이대** | 대학가 | 서아(Ep.1) 라인, 캠퍼스 데이트 |
| **홍대입구** | 번화가 | 번화가 외출, 버스킹·놀이터 밤 씬 |
| **합정** | 카페·한강 | 잔잔한 대화, 한강공원 감정 씬 |
| **성수** | 감성 카페·핫플 | 지우(Ep.3) 데이트, 분위기 좋은 씬 |
| **문래** | 창작촌·골목·조용한 술집 | 도윤(Ep.2) 술자리 밤, 진솔한 대화 |
| **건대입구** | 먹자골목 | 시끌벅적 술자리, 코미디 씬 |
| **강남** | 세련·번화 | 가끔 가는 화려한 외출, 민결(Ep.4) 접점 |

> 운용: 거점 **잠실**에서 출발해, 에피소드마다 이동 가능한 역이 점점 열린다(잠금 해제식).
> 처음엔 잠실·신촌 정도만, 진행하며 홍대입구·성수·문래·강남이 열려 세계가 넓어지는 감각을 줌.

## 5. 진행 구조 — 시간순 옴니버스 + 가벼운 연속성

병렬 루트가 아니라, **시간순으로 흘러가는 옴니버스(에피소드 단위)** 구조.
각 에피소드는 한 인물/한 관계를 중심으로 어느 정도 자기완결적이되,
**이전 에피소드를 언급하거나 일부 인물이 재등장**하는 정도의 가벼운 연속성을 가진다.

```
[프롤로그] 오픈챗 첫 입장 — 채팅 UI 튜토리얼, 도윤과 게임방에서 첫 인연

[Ep.1] 서아 — "빠르게 불타는 썸"
        └ 대학 연합 오픈챗에서 직진. 짧고 강렬하게 타올랐다 식는 한 챕터.

[Ep.2] 도윤 — "그 술자리 밤"
        └ 현실 첫 만남, 무너진 도윤을 수습 → 조력자로 굳어짐.
          (Ep.1 서아 이야기를 도윤이 술자리에서 언급 → 연속성)

[Ep.3] 지우 — "천천히 진짜가 되는"
        └ 보드게임카페에서 게임 친구로 시작. 가장 잔잔하고 깊은 에피소드.
          (도윤이 옆에서 조언/참견 → 인물 재등장)

[Ep.4] 민결 — "싸우다 친해진"
        └ 오픈챗 말다툼에서 시작. 도윤의 과거와 얽히는 반전.
          (이전 에피소드들의 선택/소문이 회수됨)

[에필로그] 그동안의 진심 게이지 총합에 따라 마무리 분기.
```

### 연속성 운용 규칙 (가볍게)
- **도윤은 모든 에피소드를 관통하는 상수.** 매 에피소드에 등장해 연속성을 묶는 끈 역할.
- 각 에피소드는 새로 시작해도 이해되게 하되, **앞 에피소드의 사건을 대사로 한두 번 언급**.
- 재등장은 **소수만** — 예: Ep.1 서아가 Ep.4에 짧게 카메오, 도윤은 상시.
- 무겁게 다 엮으려 하지 말 것. "아 저번 그 사람" 정도의 가벼운 콜백이 목표.

### 엔딩 분기 예시
- 각 에피소드마다 **소분기 결말**(굿/씁쓸 엔딩)이 존재.
- 전체 **에필로그**는 누적 진심 게이지로 결정:
  - **진엔딩** (진심↑): 한 사람과 진짜 관계로 성장.
  - **어장 엔딩** (호감↑·진심↓): 모두에게 잘했지만 아무도 남지 않음.
  - **도윤 우정 엔딩**: 연애와 별개로 조력자 라인 완수 시 해금.

## 6. 진심 게이지 설계 (수치 가이드)

각 히로인마다 두 수치를 따로 관리:
- **호감도(Like)**: 0~100. 대화·데이트·리액션으로 상승. 표면 관계.
- **진심(Sincerity)**: 0~100. 약한 모습 보이기, 진솔한 답, 약속 지키기로만 상승. 가벼운 멘트로는 안 오름.

루트 진입 조건 예시:
- 진엔딩: 진심 ≥ 70 **그리고** 호감 ≥ 60
- 굿엔딩: 진심 ≥ 50
- 어장 엔딩: 호감 ≥ 70 이면서 진심 < 30 인 상대가 2명 이상

> 의도: "잘 보이기"는 쉽지만 "진심 주기"는 비용이 든다는 걸 시스템으로 체감.

## 7. Ren'Py 화면 구조 (개발 메모)

- `script.rpy` — 메인 흐름, 챕터 라벨(`label chapter1:` 등).
- `characters.rpy` — Character() 정의, 말풍선 색/이름 색 분리.
- `screens_chat.rpy` — **카톡 UI 커스텀 screen** (말풍선 imagebutton/textbutton, 프사 표시). 이 게임의 핵심 화면.
- `screens_map.rpy` — **2호선 노선도 맵 화면**. 역 imagebutton을 노선도 위에 배치, 잠금/해금 상태 표시, 선택 시 해당 역 라벨로 점프.
- `variables.rpy` — 각 히로인 `like_`, `sincerity_` 변수 + 역 해금 플래그(`unlock_seongsu` 등) default 선언.
- `routes/` — 에피소드별 스크립트 분리.
- `images/` — 배경(실제 장소·역 사진 톤 일러스트), 노선도 이미지, 입 모양/표정 차분화.

### 2호선 맵 화면 구현 방향
1. `screen subway_map():` — 노선도 배경 이미지 위에 각 역을 `imagebutton`으로 배치.
2. 역 해금 여부를 변수로 관리(`if unlock_seongsu:` → 활성, 아니면 회색/잠금 표시).
3. 역 선택 시 `Jump("station_hongdae")` 등 해당 역 라벨로 이동.
4. 에피소드 진행에 따라 해금 플래그를 켜서 갈 수 있는 역을 늘림.

### 카톡 UI 구현 방향
1. 일반 say 화면과 별개로 `screen chat_bubble(who, msg, side)` 정의.
2. side="left"(상대)/"right"(나)에 따라 정렬·색 분기.
3. 선택지는 채팅 입력창처럼 보이는 `menu` 스타일로 커스텀.
4. 프사는 작은 이미지로 말풍선 옆에.

## 8. 첫 작품 스코프 조언 (중요)

완성이 제일 어렵다. 처음부터 3루트 다 쓰려 하면 90%가 미완성으로 끝난다.

**1차 목표 = 에피소드 1개 완성 (옴니버스라 슬라이스가 깔끔함):**
- **프롤로그 + Ep.1(서아) 한 편**을 소분기 결말까지 끝까지 완성.
- 카톡 UI / 진심 게이지 / 조력자 상담 시스템을 이 한 편에서 전부 검증.
- 옴니버스라 한 편이 완결되면 그 자체로 "플레이 가능한 작품"이 됨 → 동기 유지에 유리.
- 이게 굴러가면 Ep.2~4는 같은 틀(배경·UI·시스템 재사용)로 빠르게 증식.
